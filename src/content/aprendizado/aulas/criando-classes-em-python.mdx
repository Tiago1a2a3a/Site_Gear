# O que você vai aprender

Ao terminar esta aula, você será capaz de criar seus próprios tipos de objetos, guardar dados neles e organizar comportamentos em métodos. Vamos construir uma pequena classe `Robo` passo a passo.

> **Ideia central:** uma classe é um molde; um objeto é uma instância criada a partir desse molde.

## Antes de começar

Você precisa ter Python 3 instalado e saber executar um arquivo `.py`. Se esta for sua primeira experiência com Python, comece pelo [tutorial oficial](https://docs.python.org/3/tutorial/).

No vídeo abaixo, você verá uma introdução visual à programação orientada a objetos:

## 1. Criando a primeira classe

Uma classe começa com a palavra-chave `class`. Por convenção, nomes de classes usam `PascalCase`.

```python
class Robo:
    pass


wall_e = Robo()
print(type(wall_e))
```

`Robo` é a classe. `wall_e` é um objeto, também chamado de instância. O `pass` significa que ainda não definimos nenhum comportamento.

## 2. Atributos e o método `__init__`

Objetos geralmente precisam nascer com um estado inicial. O método especial `__init__` é executado automaticamente quando criamos uma instância.

```python
class Robo:
    def __init__(self, nome, bateria=100):
        self.nome = nome
        self.bateria = bateria


wall_e = Robo("Wall-E", 80)
print(wall_e.nome)      # Wall-E
print(wall_e.bateria)   # 80
```

### O que significa `self`?

`self` representa o próprio objeto. Quando escrevemos `self.nome`, estamos guardando o nome dentro daquela instância específica.

```python
robo_a = Robo("Atlas", 100)
robo_b = Robo("Luna", 60)

print(robo_a.nome)  # Atlas
print(robo_b.nome)  # Luna
```

Os dois objetos usam o mesmo molde, mas possuem dados independentes.

## 3. Criando métodos

Métodos são funções que pertencem à classe. Eles descrevem ações que o objeto sabe executar.

```python
class Robo:
    def __init__(self, nome, bateria=100):
        self.nome = nome
        self.bateria = bateria

    def apresentar(self):
        return f"Olá! Eu sou {self.nome}."

    def mover(self, distancia):
        consumo = distancia * 2
        if consumo > self.bateria:
            return "Bateria insuficiente."

        self.bateria -= consumo
        return f"{self.nome} moveu {distancia} metros."


robo = Robo("Luna", 80)
print(robo.apresentar())
print(robo.mover(10))
print(f"Bateria restante: {robo.bateria}%")
```

Observe que o método `mover` altera o estado do objeto. Essa é uma das grandes vantagens de agrupar dados e comportamentos na mesma classe.

## 4. Protegendo regras do objeto

Uma classe pode impedir estados inválidos. Neste exemplo, a bateria nunca fica abaixo de zero nem acima de cem.

```python
class Robo:
    def __init__(self, nome, bateria=100):
        self.nome = nome
        self.bateria = max(0, min(bateria, 100))

    def recarregar(self, porcentagem):
        if porcentagem < 0:
            raise ValueError("A recarga não pode ser negativa.")

        self.bateria = min(100, self.bateria + porcentagem)


robo = Robo("Atlas", 30)
robo.recarregar(50)
print(robo.bateria)  # 80
```

## 5. Herança: reaproveitando uma classe

Herança permite criar uma classe especializada a partir de outra. Um `RoboAspirador` é um tipo de `Robo`, mas possui uma ação extra.

```python
class Robo:
    def __init__(self, nome):
        self.nome = nome

    def apresentar(self):
        return f"Eu sou {self.nome}."


class RoboAspirador(Robo):
    def aspirar(self):
        return f"{self.nome} está limpando o chão."


robo = RoboAspirador("Dusty")
print(robo.apresentar())
print(robo.aspirar())
```

Use herança quando existe uma relação clara de “é um”. Quando a relação for “tem um”, composição costuma ser mais simples.

## 6. Composição: objetos trabalhando juntos

Um robô pode ter um sensor. O robô não é um sensor; ele **tem** um sensor.

```python
class SensorDistancia:
    def __init__(self, limite):
        self.limite = limite

    def detectou_obstaculo(self, distancia):
        return distancia <= self.limite


class Robo:
    def __init__(self, nome, sensor):
        self.nome = nome
        self.sensor = sensor

    def decidir(self, distancia):
        if self.sensor.detectou_obstaculo(distancia):
            return "Parar e desviar."
        return "Continuar em frente."


sensor = SensorDistancia(limite=20)
robo = Robo("Scout", sensor)
print(robo.decidir(15))  # Parar e desviar.
```

## 7. Um projeto completo

Agora vamos juntar os conceitos em um pequeno sistema de tarefas.

```python
class Tarefa:
    def __init__(self, titulo):
        self.titulo = titulo
        self.concluida = False

    def concluir(self):
        self.concluida = True

    def __str__(self):
        status = "x" if self.concluida else " "
        return f"[{status}] {self.titulo}"


class ListaDeTarefas:
    def __init__(self):
        self.tarefas = []

    def adicionar(self, titulo):
        self.tarefas.append(Tarefa(titulo))

    def concluir(self, indice):
        self.tarefas[indice].concluir()

    def mostrar(self):
        for indice, tarefa in enumerate(self.tarefas, start=1):
            print(indice, tarefa)


lista = ListaDeTarefas()
lista.adicionar("Estudar classes")
lista.adicionar("Criar um projeto")
lista.concluir(0)
lista.mostrar()
```

Saída esperada:

```text
1 [x] Estudar classes
2 [ ] Criar um projeto
```

## Desafio prático

Crie uma classe `ContaBancaria` com:

1. `titular` e `saldo` no construtor;
2. um método `depositar(valor)`;
3. um método `sacar(valor)` que bloqueie saldo insuficiente;
4. um método `__str__` que mostre o titular e o saldo formatado.

Depois, crie duas contas e prove que o saldo de uma não altera o saldo da outra.

<details>
<summary>Uma possível solução</summary>

```python
class ContaBancaria:
    def __init__(self, titular, saldo=0):
        self.titular = titular
        self.saldo = saldo

    def depositar(self, valor):
        if valor <= 0:
            raise ValueError("O depósito deve ser positivo.")
        self.saldo += valor

    def sacar(self, valor):
        if valor > self.saldo:
            raise ValueError("Saldo insuficiente.")
        self.saldo -= valor

    def __str__(self):
        return f"{self.titular}: R$ {self.saldo:.2f}"
```

</details>

## Recapitulando

- `class` define um novo tipo de objeto.
- `__init__` prepara o estado inicial.
- `self` aponta para a instância atual.
- Atributos guardam dados; métodos executam comportamentos.
- Herança especializa uma classe existente.
- Composição conecta objetos que trabalham juntos.
